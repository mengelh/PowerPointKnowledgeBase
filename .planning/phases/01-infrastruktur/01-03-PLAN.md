Two tasks: update .gitignore to block local secrets files, and verify/encrypt vault.yml.

**1. Update .gitignore** — add the following entries (append to existing .gitignore if one exists,
create if not):

```
# Ansible vault password files — NEVER commit these
.vault_pass
.vault-pass
vault_pass.txt
*.vault-pass

# Local override files
*.local
.env.local
ansible/.env

# Python
__pycache__/
*.pyc
.venv/
venv/

# Ansible temp files
*.retry
```

**2. Update ansible/inventory/group_vars/all.yml** — add SSH args for Semaphore runner if not
already present. Append to the file:

```yaml
# Semaphore runner connection settings (test VM only)
ansible_ssh_common_args: "-o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null"
```

Note: StrictHostKeyChecking=no is acceptable for the test VM. Production deployments should use
known_hosts or certificate-based host verification.

**3. Verify vault.yml is encrypted** — this is a CHECK task, not a write task:

```bash
head -1 ansible/inventory/group_vars/all/vault.yml
```

If the output is "$ANSIBLE_VAULT;1.1;AES256" → vault is encrypted, proceed.
If the output is "# THIS FILE MUST BE ENCRYPTED" or "vault_authentik_secret_key:" → NOT encrypted.

If NOT encrypted:
1. Generate real secrets (do this locally, do NOT commit these values):
   ```bash
   openssl rand -hex 32    # vault_authentik_secret_key
   openssl rand -base64 24 | tr -d '/'  # vault_pg_password
   ```
2. Edit ansible/inventory/group_vars/all/vault.yml and replace REPLACE_BEFORE_ENCRYPT with real values
3. Encrypt: `ansible-vault encrypt ansible/inventory/group_vars/all/vault.yml --ask-vault-pass`
4. Verify: `head -1 ansible/inventory/group_vars/all/vault.yml` → must show $ANSIBLE_VAULT header
5. Run: `git grep -i "REPLACE_BEFORE_ENCRYPT\|openssl\|hex32" ansible/` → must return empty

**4. Run the secret hygiene check**:
```bash
git grep -i "password\|secret_key\|pg_pass" -- ansible/ | grep -v "vault_" | grep -v "\.vault"
```
Expected output: no lines (all credential references use vault_ variables).

If any plain-text secrets are found: STOP — fix before committing. This is a P0 security issue.
